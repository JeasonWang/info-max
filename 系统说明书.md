# 信息达人系统说明书

## 1. 系统概述

信息达人（`Info Max`）是一个面向 Web / H5 的热点信息聚合前端系统，对接后端 `Info_aggregation` 服务，用于展示多渠道热点、科技、AI 与其他分类信息。系统目标是让用户进入首页后，第一时间看到最重要的热点内容，而不是先理解系统结构。

## 2. 系统组成

### 前端

- 技术栈：`Vue 3`、`TypeScript`、`Vite`
- 目录位置：`Info_max/`
- 主要职责：
  - 展示热点列表与详情
  - 提供分类、渠道、关键词筛选
  - 展示统计、时间线、收藏状态
  - 触发后端渠道爬取

### 后端

- 技术栈：`Python`、`FastAPI`
- 目录位置：`Info_aggregation/`
- 主要职责：
  - 聚合多渠道爬虫数据
  - 提供分类、渠道、列表、详情、统计接口
  - 提供手动触发爬取能力

## 3. 首页设计原则

首页遵循“主内容优先，辅助功能折叠”的原则：

- 默认优先展示：
  - 热点列表
  - 筛选区
- 默认弱化并折叠：
  - 产品说明
  - 我的阅读状态
  - 统计面板
  - 时间线
  - 渠道运维

这样做的原因是：用户进入首页时的核心任务是“看热点”，而不是“理解系统”或“管理渠道”。

## 4. 主要功能说明

### 4.1 热点列表

- 展示后端返回的信息列表
- 支持分页切换
- 支持收藏
- 支持跳转详情
- 支持打开原始来源

### 4.2 筛选区

- 默认折叠，减少首页占位
- 点击“展开筛选”后显示分类、渠道和关键词
- 分类：按钮式 Tab，点击即筛选，再次点击当前项可取消
- 渠道：按钮式 Tab，点击即筛选，再次点击当前项可取消
- 关键词：自动搜索，支持清空和重置
- 点击筛选后面板会自动收起，避免遮挡列表阅读

### 4.3 详情页

- 展示标题、时间、主体、地点、指标
- 支持正文格式化展示
- 支持收藏
- 支持分享
- 支持打开原文

### 4.4 更多内容

更多内容为右侧抽屉，收纳如下模块：

- 我的阅读状态
- 渠道运维
- 配置管理入口

### 4.5 渠道运维

- 展示渠道名称、地址、编码、爬取间隔
- 支持手动触发后端爬取
- 适合运营和调试场景，不适合作为首页主区域

### 4.6 配置管理

- 提供分类配置页面
- 提供渠道配置页面
- 支持新增和编辑
- 渠道支持切换启用 / 停用状态
- 配置入口位于“更多内容”抽屉中，面向管理人员使用

## 5. 交互说明

### 首页交互

- 进入页面后先看到筛选器和热点列表
- 筛选器默认收起，不固定在页面上，不影响向下浏览
- 点击 Tab 立即筛选，筛选完成后自动收起
- 点击“更多内容”可打开右侧抽屉查看辅助功能

### 收藏交互

- 收藏状态保存在浏览器本地 `localStorage`
- 同一浏览器下刷新后依然保留

### 分享交互

- 优先调用浏览器原生分享能力
- 如果当前环境不支持，则复制详情页链接

## 6. 运行说明

### 前端启动

目录：`Info_max/`

```bash
npm install
npm run dev -- --host 127.0.0.1 --port 5173
```

访问地址：

- `http://127.0.0.1:5173/`

### 后端启动

目录：项目根目录

如果未创建虚拟环境：

```bash
python3 -m venv Info_aggregation/.venv
Info_aggregation/.venv/bin/python -m pip install -r Info_aggregation/requirements.txt
```

启动命令：

```bash
Info_aggregation/.venv/bin/python Info_aggregation/main.py
```

访问地址：

- API 服务：`http://127.0.0.1:8000/`
- API 文档：`http://127.0.0.1:8000/docs`

## 7. 接口对接说明

当前前端已对接以下接口：

- `GET /api/categories`
- `GET /api/channels`
- `GET /api/infos`
- `GET /api/infos/{id}`
- `GET /api/stats`
- `POST /api/crawl/trigger`
- `GET /api/admin/categories`
- `POST /api/admin/categories`
- `PUT /api/admin/categories/{category_id}`
- `GET /api/admin/channels`
- `POST /api/admin/channels`
- `PUT /api/admin/channels/{channel_id}`

## 8. 异常与兼容处理

- 列表加载失败：展示错误提示和重试按钮
- 详情加载失败：展示错误提示和重试按钮
- 空数据：展示空态提示
- 长链接：已做自动换行，避免超出卡片边界
- 窄屏场景：页面自动切换为单栏布局
- 后端每天会清理两周前创建的历史数据
